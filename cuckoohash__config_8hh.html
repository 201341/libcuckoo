◆ LIBCUCKOO_DEFAULT_MINIMUM_LOAD_FACTOR

constexpr double LIBCUCKOO_DEFAULT_MINIMUM_LOAD_FACTOR = 0.05

The default minimum load factor that the table allows for automatic expansion. It must be a number between 0.0 and 1.0. The table will throw libcuckoo_load_factor_too_low if the load factor falls below this value during an automatic expansion.

◆ LIBCUCKOO_LOCK_ARRAY_GRANULARITY

constexpr size_t LIBCUCKOO_LOCK_ARRAY_GRANULARITY = 0

On a scale of 0 to 16, the memory granularity of the locks array. 0 is the least granular, meaning the array is a contiguous array and thus offers the best performance but the greatest memory overhead. 16 is the most granular, offering the least memory overhead but worse performance.

◆ LIBCUCKOO_NO_MAXIMUM_HASHPOWER

constexpr size_t LIBCUCKOO_NO_MAXIMUM_HASHPOWER = 0

An alias for the value that sets no limit on the maximum hashpower. If this value is set as the maximum hashpower limit, there will be no limit. Since 0 is the only hashpower that can never occur, it should stay at 0. This is also the default initial value for the maximum hashpower in a table.
